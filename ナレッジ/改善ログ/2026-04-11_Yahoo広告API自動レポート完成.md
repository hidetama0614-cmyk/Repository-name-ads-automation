# Yahoo広告API自動レポート 完成記録

## 概要
LINEヤフー広告（検索広告）の実績データをGoogleスプレッドシートに自動出力する仕組みを構築した。

**完成日**: 2026-04-11  
**ファイル**: `マーケティング/yahoo_report.py`  
**自動実行**: `.github/workflows/daily_yahoo_report.yml`（毎日9時JST）

---

## 出力内容

| タブ名 | 内容 | 件数目安 |
|---|---|---|
| 【添付】N月Yahoo!広告 | キャンペーン実績（インプレッション・費用・CV等） | キャンペーン数分 |
| 【添付】N月Yahoo!広告_CV内訳 | キャンペーン × コンバージョン名別のCV数 | CV設定数 × キャンペーン数 |

毎月自動でタブが新規作成される（5月→「5月Yahoo!広告」など）。  
月初1日はデータなしのためスキップ。実データは翌2日から。

## 詰まったポイントと解決策

### 1. 正しいAPIエンドポイントの発見
- **誤**: `ads.yahoo.co.jp/api/v13/` → 404
- **正**: `ads-search.yahooapis.jp/api/v19/ReportDefinitionService`

### 2. x-z-base-account-id ヘッダーが必須
全リクエストに `x-z-base-account-id: 1001894160` が必要。  
アカウントIDは2種類ある（ベースアカウント: 1001894160、キャンペーンアカウント: 1003214）。

### 3. CONVERSION_NAME は集計指標と混在不可（RL001）
インプレッション・費用・CTRなどと同じレポートに入れるとエラー。  
**解決**: キャンペーン実績とCV内訳を別々のレポートジョブで取得し、別タブに出力。  
CONV_RATE（コンバージョン率）もCONVERSION_NAMEと組み合わせ不可。

### 4. /get エンドポイントでselector/reportJobIdが無効（v19仕様）
`selector`や`reportJobId`を渡すと400エラー。  
`accountId`のみで全ジョブ一覧を取得し、クライアント側でjob_idを探す方式で解決。

### 5. スプレッドシートへの書き込み権限（403）
スプレッドシートをサービスアカウント（`ads-automation-sheets@ads-automation-491012.iam.gserviceaccount.com`）に「編集者」で共有する必要あり。

## GitHub Secrets（必要なもの）

| Secret名 | 内容 |
|---|---|
| YAHOO_ADS_CLIENT_ID | LINEヤフー広告アプリのクライアントID |
| YAHOO_ADS_CLIENT_SECRET | クライアントシークレット |
| YAHOO_ADS_REFRESH_TOKEN | リフレッシュトークン |
| YAHOO_ADS_ACCOUNT_ID | キャンペーンアカウントID（1003214） |
| GOOGLE_SERVICE_ACCOUNT_JSON | Googleサービスアカウントのキー（JSON全体） |
