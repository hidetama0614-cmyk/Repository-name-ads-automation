# このリポジトリについて

通信販売（化粧品・健康食品）事業の広告運用自動化・商品企画のためのスクリプト・エージェント定義集。

## フォルダ構成
- `マーケティング/` … Google Ads / Yahoo広告 / Meta広告のレポート自動取得・分析・Slack通知スクリプト（`.env`のAPIキー、`service_account.json`が必要。GitHub Actionsで定期実行）
- `商品企画/` … 競合分析〜自社商品企画のスクリプト・エージェント定義（詳細は次項）
- `技術（ツール開発・自動化）/` … 認証・共通設定
- `claude/agents/` … 各種専門エージェントの役割定義（Markdown、参照用）
- `ナレッジ/` … 改善ログ等のドキュメント

## 商品企画エージェント（重要）

ユーザーから「商品テーマ」「カテゴリ」「競合商品ページのURL（2〜5件）」が渡されたら、
`claude/agents/product-planner.md` の役割定義に沿って、以下を行う。

1. 各URLをWebFetchで取得し、競合分析（商品特長・メイン訴求・選ばれる理由・不満な声など）を行う
2. 市場調査・自社商品コンセプト（企画サマリー、参入余地、メイン訴求、商品の強み、商品形状、想定売り場、売価/容量、キー成分、ボトルネックとそれを超える策、解雇（解約）条件）を設計する
3. 分析結果をJSONにまとめ、`商品企画/generate_concept_sheet.py` を実行して `.xlsx` を生成する
   ```
   python 商品企画/generate_concept_sheet.py <入力JSON>
   ```
   （入力JSONの形式はスクリプト冒頭のdocstring参照。外部LLM API・Google認証は不要）
4. 生成物は `商品企画/output/`（Gitにはコミットしない）に保存し、`SendUserFile` 等でユーザーに渡す

詳しい使い方は `商品企画/README_商品企画エージェント.md` を参照。

## 機密情報の扱い
`.env` / `service_account.json` はコミットしない（`.gitignore`で除外済み）。マーケティング系スクリプトの実行にはこれらローカルの認証情報が必要（商品企画エージェントには不要）。
